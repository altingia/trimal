437  /**
438  \brief Method to assign sequences to clusters.\n
439  Clusters are calculated following this flow:
440  - Select the longest sequence and
441  use it as representative of the first cluster.\n
442  - Using the previously computed identity values,
443  check if second longest sequence should be part of
444  cluster of first sequence.
445  This is computed using the identity value
446  in comparison with a threshold.\n
447  - If the second sequence should be part of the existing cluster, we add it.
448  Otherwise we create a new cluster and
449  use this sequence as representative for this new cluster.\n
450  - Continue with the rest of sequences,
451  comparing them with the representatives of existing clusters.
452  If a sequence can pertain to more than one cluster,
453  we choose the one that maximizes the identity value
454  with the representative sequence.
455  \param maximumIdent
456  Identity threshold used to decide if a sequence should be part of
457  a cluster or create a new one.
458  \return Vector that contains the clustering info.\n
459  The first item in the vector contains it's size.
460  */
